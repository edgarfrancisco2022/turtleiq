---
title: Building a System You Will Actually Use
date: 2026-02-20
---

Most productivity systems fail for the same reason: they are optimized for the state of mind in which you designed them, not the state of mind in which you need to use them.

You design the system on a Saturday afternoon, energized, with a clear head and good intentions. You need to use it on a Tuesday evening, tired, with fifteen other things competing for your attention. The system that seemed elegant on Saturday becomes friction on Tuesday. So you skip it. Then you skip it again. Then you stop thinking of yourself as someone who uses that system.

I have built and abandoned many learning systems. Elaborate Notion databases, nested folder hierarchies, color-coded tagging schemas. Each one felt like progress when I built it. None of them survived contact with real life.

The insight that changed my approach was simple: the cost of the input determines whether the system gets used.

If adding something to the system takes more than a minute or two, I will defer it. If I defer it, I will forget about it. If I forget about it, the system is empty. An empty system is useless. A useless system gets abandoned.

So the primary design constraint for TortugaIQ was low input friction. Adding a concept should be fast. The mandatory fields are minimal: a name and a subject. Everything else — topic, subtopic, tags, notes — is optional and can be added later.

The secondary constraint was that the system should be useful at scale. A lot of systems are fine when small and become unwieldy as they grow. The organization scheme that worked for fifty items fails at five hundred. I wanted something that would remain navigable as the number of concepts grew into the hundreds or thousands.

The answer to both constraints was the concept itself: a small, named, atomic piece of knowledge. Easy to create. Easy to filter. Easy to find. Easy to review.

What I have learned from building and using this system is that simplicity is not a compromise. It is the feature. The system I will actually use is not the most powerful one I can imagine. It is the most frictionless one that does the job.

That is a harder design problem than it sounds.
