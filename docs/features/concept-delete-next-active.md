# Next concept to be marked as active when deleting a concept from the Library view

On the Library view, when we delete a concept, what happens is that the currently active concept element default to the first element on the list.

This seems counterinuitive.

My initial intuition is that when we delete a concept element either the concept element which is just before the element being deleted or the element just after the element being deleted should be marked as active, so that the user can continue navigating from the place where the deleted concept was, rather than being "redirected" to the beginning of the list.

This functionality might have additional edge cases Im not aware of.

Please create a plan to address this issue, it does not have to be the way I am suggesting, thats just my non-expert intutition, please assume the role of a UI/UX expert feeling free to ignore my suggestion and provide a better one if there is, otherwise please based on my idea make necessary adjustments as needed to make it consistent across possible edge cases.
