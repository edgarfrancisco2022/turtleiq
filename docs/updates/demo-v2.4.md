# Updates for the demo-v2.4 version

The main update for this version upgrade is that we are introducing blogging functionality for the site.

### Blogging functionality should exist in the /notes path

- The design of this blog should be simple.
- For now we do not want any type of complexity (like table of contents, or search functionalities).
- All we want is to display the blog entries in the /notes in newest first order so that the user may click and navigate to any of them.
- That being said the design of this page and subpages should look professional, trendy, and sleek. It should be appealing if it is simple.
- I feel for now we do not need any fancy metadata for the blog entries, we only need the date and the title of the blog entry.
- The order of the blog entries should be in newest first.
- We want to use markdown to create these blog entries, so a solution similar to the markdown editors we already have in the Concept view might be acceptable, otherwise feel free to think of best solution. In any case, we want to use markdown to write the blog entries, and just render the markdown when the user is viewing one of the entries.
- My initial idea was to add blog entries by adding markdown files to the github codebase and reploy the app, but this feels kind of a lazy solution.
- My second idea is to have within the app an option to add blog entries, however this option would have to be password protected or something so that only I can add entries, or anycase later when we add more robust security potentially admin users.
- Please provide feedback on what you think is the best idea out of these two to implement blogging, or if you think there is a better or more simple idea.
- Please provide what you think would be the best workflow to add blog entries.
- I will leave up the design of the /notes page as well as the blog entry pages, or any other parts of the workflow, to your own expertise, it should be simple, professional, trendy, and sleek, and match with the design of the rest of the application, in particular it should match with the design of the landing page.
